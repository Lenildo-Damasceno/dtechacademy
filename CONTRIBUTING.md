# 🤝 Como Contribuir com o Projeto

Este guia explica como adicionar um colaborador ao repositório para que ele possa editar o projeto diretamente no GitHub.

---

## 👥 Adicionando um Colaborador (Amigo) ao Repositório

Para dar acesso ao seu amigo para que ele possa editar o projeto, siga os passos abaixo:

### Passo 1 – Acesse as configurações do repositório

1. Abra o repositório no GitHub: `https://github.com/Lenildo-Damasceno/inovatech`
2. Clique na aba **"Settings"** (Configurações), no menu superior do repositório.

### Passo 2 – Vá em "Collaborators"

1. No menu lateral esquerdo, clique em **"Collaborators"** (ou "Colaboradores").
2. O GitHub pode pedir para você confirmar sua senha ou autenticação de dois fatores.

### Passo 3 – Convide o colaborador

1. Clique no botão **"Add people"** (Adicionar pessoas).
2. Digite o **nome de usuário do GitHub** do seu amigo ou o **e-mail** dele.
3. Selecione o usuário correto na lista de sugestões.
4. Clique em **"Add [nome]"** para enviar o convite.

### Passo 4 – Seu amigo aceita o convite

1. Seu amigo receberá um **e-mail de convite** do GitHub.
2. Ele precisa clicar no link do e-mail para **aceitar o convite**.
3. Após aceitar, ele terá acesso para clonar, editar e enviar alterações ao repositório.

---

## 🛠️ Como Contribuir com o Código (para colaboradores)

Após ser adicionado como colaborador, siga este fluxo para editar o projeto:

### Clonar o repositório

```bash
git clone https://github.com/Lenildo-Damasceno/inovatech.git
cd inovatech
```

### Criar uma branch para suas alterações

```bash
git checkout -b minha-feature
```

### Fazer alterações e salvar

```bash
git add .
git commit -m "Descrição clara do que foi alterado"
```

### Enviar para o GitHub

```bash
git push origin minha-feature
```

### Abrir um Pull Request

1. Acesse o repositório no GitHub.
2. Clique em **"Compare & pull request"**.
3. Descreva as alterações feitas e clique em **"Create pull request"**.
4. Aguarde a revisão e aprovação do dono do repositório.

---

## 📋 Boas Práticas

- Sempre crie uma **nova branch** para cada nova funcionalidade ou correção.
- Escreva **mensagens de commit claras** e descritivas.
- Teste suas alterações antes de enviar o Pull Request.
- Mantenha o código organizado seguindo o padrão já existente no projeto.
